Important : le modèle évalué dans le WF est le modèle final entraîné sur 80% des données — pas un modèle réentraîné à chaque fold.
→ Les folds 1-2 sont dans le train set du modèle final → leurs Sharpes sont moins informatifs (le modèle a "vu" ces données).

Pourquoi 5 folds ? test_frac=10% × 5 = 50% du dataset testé OOS, chaque fold ≈ 400 jours (>1 an de crypto) = suffisant pour voir au moins un cycle de vol.

Gap de 5 jours : les features rolling (corr_21d, mom_90d) utilisent des données du passé récent. Sans gap, les derniers jours du train set "contamineraient" le calcul des features du premier jour du test set.

À la fin des 512 steps : le buffer contient 512 tuples (obs, action, reward, done, log_prob, value, h, c). Le LSTM n'est pas mis à jour pendant la collecte — il est en mode inférence uniquement.
G
Calcul des avantages GAE (Generalized Advantage Estimation) POST-COLLECTE, PRÉ-UPDATE
δ_t = r_t + γ · V(s_{t+1}) − V(s_t)   // TD error au step t
A_t = δ_t + (γ·λ)·δ_{t+1} + (γ·λ)²·δ_{t+2} + ...   // avantage GAE, calculé en remontant dans le temps
Returns_t = A_t + V(s_t)   // cible pour le critic

Paramètres : γ=0.995 (discount), λ=0.95 (GAE lambda)
A_t > 0 : l'action était meilleure qu'attendu → augmenter sa proba
A_t < 0 : l'action était pire qu'attendu → diminuer sa proba
S
Shuffle des séquences — le buffer de 512 steps est découpé en 512 / 64 = 8 mini-batches de 64 steps consécutifs. Les séquences sont mélangées (pas les steps à l'intérieur d'une séquence — l'ordre temporel doit être préservé pour le LSTM).
①
Re-forward pass LSTM sur les 64 steps LSTM MIS À JOUR ICI
On récupère l'état LSTM h_0, c_0 sauvegardé au début de la séquence dans le buffer. On rejoue les 64 steps dans le LSTM pour obtenir les h_t actuels (avec les poids courants du réseau, pas ceux de la collecte).

Pourquoi re-forward et pas utiliser les h_t du buffer ?
Parce que les poids LSTM ont changé depuis la collecte (mises à jour précédentes). Les h_t du buffer sont "obsolètes" — on recalcule avec les poids actuels.
②
Forward pass Actor + Critic
Pour chaque step t dans le mini-batch :
  → Actor : h_t → log_prob(action_t) avec les poids courants
  → Critic : h_t → V_new(s_t) avec les poids courants
③
Calcul de la loss PPO
ratio_t = exp(log_prob_new_t − log_prob_old_t)   // rapport des probas nouvelle/ancienne politique

L_CLIP = −mean[ min( ratio·A_t , clip(ratio, 1−ε, 1+ε)·A_t ) ]   // ε=0.2 (clip_range)
// Empêche la politique de trop changer d'un coup → stabilité

L_VALUE = 0.5 · mean[ (V_new_t − Returns_t)² ]   // MSE critic

L_ENTROPY = −ent_coef · mean[ H(π) ]   // ent_coef=0.01, encourage l'exploration

L_TOTAL = L_CLIP + 0.5 · L_VALUE − L_ENTROPY   // loss finale
④
Backward pass — BPTT sur 64 steps DESCENTE DE GRADIENT
L_TOTAL.backward() — PyTorch remonte le gradient à travers :
  → MLP Critic (couches denses)
  → MLP Actor (couches denses)
  → LSTM sur les 64 steps (Backpropagation Through Time tronquée)

À chaque step t du LSTM, le gradient est multiplié par les forget gates f_t. Si f_t ≈ 1 → le gradient passe bien (mémoire longue apprise). Si f_t ≈ 0 → le gradient disparaît (oubli volontaire). C'est le mécanisme clé qui évite le vanishing gradient.
⑤
Adam optimizer — mise à jour des poids
optimizer.step() met à jour tous les paramètres simultanément :
  → Poids LSTM : W_i, W_f, W_g, W_o, b_i, b_f, b_g, b_o (~330k params)
  → Poids Actor MLP (~16k params)
  → Poids Critic MLP (~16k params)

Adam maintient une moyenne mobile des gradients (momentum) et une moyenne mobile du carré des gradients (RMSProp) → learning rate effectif adaptatif par paramètre. Avec lr=1e-4, les poids bougent doucement → stabilité LSTM.
✓
Gradient clipping — norme des gradients clippée à 0.5 (max_grad_norm SB3 défaut).
Évite les exploding gradients qui peuvent destabiliser le LSTM si un reward exceptionnel crée un gradient anormalement grand.
→
Early stopping par KL divergence — si la divergence KL entre ancienne et nouvelle politique dépasse target_kl (0.1 par défaut SB3), l'epoch s'arrête prématurément. Évite que 10 epochs de mise à jour ne dégrade trop la politique.
→
Buffer jeté — les 512 steps collectés et les avantages calculés sont supprimés. On recommence une nouvelle collecte avec la politique mise à jour. C'est ce qui fait de PPO un algorithme on-policy : il ne réutilise jamais les transitions passées (contrairement à SAC ou DQN qui sont off-policy).

Intérieur de la cellule LSTM — les 4 portes (gates)
Chaque step t : x_t (60 dim) + h_{t-1} (256 dim) sont concaténés → vecteur 316 dim. Multiplié par 4 matrices de poids → 4 gates qui contrôlent la mémoire.
🔴 Forget Gate f_t — Qu'est-ce qu'on oublie ?
f_t = σ( W_f · [h_{t-1}, x_t] + b_f )
sortie ∈ [0, 1] par dimension
f=0 → efface tout de c_{t-1}.  f=1 → garde tout.
Concrètement : le marché passe de bull à bear → la gate efface progressivement le contexte "on est en tendance haussière".
🟢 Input Gate i_t + g_t — Quoi mémoriser ?
i_t = σ( W_i · [h_{t-1}, x_t] )  → "à quel point"
g_t = tanh( W_g · [h_{t-1}, x_t] ) → "quelle valeur"
Les deux ensembles décident ce qui entre dans la mémoire longue.
Concrètement : RSI monte à 92 + volume spike → mémorise "signal de surchauffe".
🟡 Cell State c_t — La mémoire longue
c_t = f_t ⊙ c_{t-1} + i_t ⊙ g_t
Peut rester stable des centaines de steps sans s'effacer. Le gradient remonte directement à travers c_t sans multiplication répétée → pas de vanishing gradient.
Concrètement : "on est en bear market depuis 6 mois, drawdown -40%".
🔵 Output Gate o_t — Quoi exposer ?
o_t = σ( W_o · [h_{t-1}, x_t] )
h_t = o_t ⊙ tanh( c_t )  → 256 dim → Actor + Critic
Filtre ce qu'on révèle de la mémoire longue à chaque step.
Concrètement : expose "contexte bear" à l'Actor seulement quand c'est pertinent pour la décision du moment.

LSTM   : 4 matrices × (60+256) × 256 + 4 × 256 biases = 330 752 params
Actor MLP : 256×64 + 64 + 64×1 + 1                  =  16 449 params
Critic MLP : 256×64 + 64 + 64×1 + 1                  =  16 449 params

8 Définitions — Step / Épisode / Buffer / Epoch
Glossaire opérationnel du training RecurrentPPO

Step

Un step correspond à une journée de trading simulée. Le modèle observe les 56 features du jour (RSI, volatilité, corrélations, etc.), décide d'une fraction d’exposition comprise entre -1 et +1, puis reçoit un reward correspondant au gain ou à la perte générée.

1 000 000 de steps représentent 1 000 000 décisions de trading simulées. Il ne s'agit pas de 1 000 000 de jours réels : les données historiques sont rejouées depuis différents points de départ à chaque épisode.

Épisode

Un épisode est une séquence complète de trading simulée. Il débute à une date aléatoire dans l'historique avec un capital initial, puis progresse jour après jour jusqu'à atteindre la fin des données ou un seuil de drawdown prédéfini (kill-switch).

Chaque épisode dure au minimum 365 steps. Plusieurs milliers d’épisodes sont parcourus au cours de l’entraînement, chacun commençant à un point différent de l’historique.

Buffer

Le buffer est une mémoire temporaire contenant 512 steps consécutifs. Pour chaque step, sont enregistrés :

  • Les observations (56 features + variables de portefeuille)
  • L’action prise
  • Le reward reçu
  • Les états LSTM (h et c)

Une fois rempli, le buffer est utilisé pour l’apprentissage, puis supprimé avant de recommencer une nouvelle phase de collecte.

Epoch

Un epoch correspond à une relecture complète du buffer. Plutôt que d'apprendre une seule fois sur les 512 steps collectés, le modèle effectue 10 passes successives sur ces mêmes données, ce qui améliore l’efficacité de l’apprentissage.

Mini-batch

Le buffer de 512 steps est divisé en 8 mini-batches de 64 steps consécutifs. Les séquences restent ordonnées dans le temps afin de préserver la cohérence des états LSTM.

Re-forward

Lors de l’apprentissage, chaque mini-batch est rejoué dans le réseau avec les poids actuels du modèle. Les états LSTM sont recalculés à partir de l’état initial sauvegardé, afin d’obtenir des probabilités d’action cohérentes avec les paramètres mis à jour.

Évaluation pendant le training

Actor : Ajuste la probabilité des actions passées en fonction de leur avantage (A_t).

Critic : Apprend à prédire la valeur future attendue depuis un état donné.
